Important contract notes
- manual generation returns generation_run_id
- GET /api/jobs/{id} includes latest_generation_run for queued/running/failed generation progress
- job detail artifact items surface backend metadata summary fields:
  artifact_role, payload_version, inputs_hash, fit_status, evidence_completeness
- ready-to-apply returns only artifact-ready NEW jobs
- GET /api/jobs/run-source-adapter is the operator capability read model for adapter-backed launch
- POST /api/jobs/run-source-adapter launches the selected adapter-backed run
- run items are normalized through core/run_items.py
- debug data in job detail is gated behind DEBUG_ENDPOINTS_ENABLED
